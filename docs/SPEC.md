## Overview

An **ai-skill** is a self-contained, portable unit of AI capability. It describes *what* a skill does, *how* it runs, and *how to measure* its quality — in a format that any AI agent framework can consume.

The format is intentionally simple. A skill is a single YAML (or JSON) file that any developer, tool, or system can read without special libraries.

### Design Goals

- **Portable** — one skill file works across all major frameworks
- **Readable** — a human should understand a skill by reading it
- **Testable** — skills include their own test cases and benchmarks
- **Composable** — skills can reference and chain other skills
- **Versioned** — skills are semantically versioned and immutable once published

## Skill File Format

A skill is defined in a file named `skill.yaml` (or `skill.json`).

### Minimal valid skill

```yaml
skill:
  id: hello-world
  version: 1.0.0
  name: Hello World
  description: Returns a greeting for a given name
  inputs:
    - name: name
      type: string
      required: true
  outputs:
    - name: greeting
      type: string
  execution:
    type: prompt
    prompt_template: "Say hello to {name} in a warm and friendly way."
```

### Full skill with all optional fields

```yaml
skill:
  # Identity
  id: summarize-document           # Unique identifier (kebab-case, required)
  version: 1.2.0                   # Semantic version (required)
  spec_version: "0.1"              # Spec version this file targets (optional)
  name: Document Summarizer        # Human-readable name (required)
  description: |                   # What this skill does (required)
    Summarizes a document into a concise summary and key bullet points.
    Works best on documents under 10,000 words.
  
  # Metadata
  author: jane-doe                 # Author username or org
  license: MIT                     # License identifier (SPDX)
  tags: [summarization, nlp, text] # Searchable tags
  homepage: https://github.com/jane/my-skills
  created_at: 2026-02-18
  updated_at: 2026-02-18

  # Input parameters
  inputs:
    - name: document               # Parameter name (required)
      type: string                 # Data type (required)
      description: The document text to summarize
      required: true               # Default: false
    
    - name: max_words
      type: integer
      description: Maximum words in the summary
      required: false
      default: 150                 # Used when parameter is not provided
      minimum: 50
      maximum: 500
    
    - name: language
      type: string
      description: Output language
      required: false
      default: english
      enum: [english, spanish, french, german, japanese]  # Allowed values

  # Output parameters
  outputs:
    - name: summary
      type: string
      description: The condensed summary
    
    - name: key_points
      type: array
      items: string
      description: List of 3-5 key bullet points
    
    - name: word_count
      type: integer
      description: Word count of the generated summary

  # How the skill executes
  execution:
    type: prompt                   # prompt | tool_call | chain | code
    
    # For type: prompt
    model_hint: any                # any | gpt-4 | claude-3 | etc (hint, not requirement)
    system_prompt: |               # Optional system prompt
      You are an expert summarizer. Be concise and accurate.
    prompt_template: |             # Required for type: prompt
      Summarize the following document in under {max_words} words.
      Language: {language}
      
      Return your response in this exact format:
      SUMMARY: [your summary here]
      KEY_POINTS:
      - [point 1]
      - [point 2]
      - [point 3]
      
      Document:
      {document}
    
    output_parser: structured      # none | structured | json | custom

  # Which frameworks this skill is compatible with
  # (auto-generated by SDK if not specified)
  compatible_with:
    - langchain
    - autogen
    - crewai
    - semantic_kernel
    - raw_api

  # Quality benchmarks (optional but recommended)
  benchmarks:
    avg_latency_ms: 1200           # Average execution time
    avg_cost_per_call_usd: 0.004   # Average cost per run
    last_evaluated: 2026-02-01
    
    # Test cases (used for automated quality checks)
    test_cases:
      - id: basic-article
        description: Summarize a short news article
        input:
          document: "Scientists have discovered a new species of deep-sea fish..."
          max_words: 50
        expected:
          summary_contains: ["scientists", "fish"]
          key_points_count_min: 2
          key_points_count_max: 5
        
      - id: multilingual
        description: Summarize in Spanish
        input:
          document: "The annual report shows strong revenue growth..."
          language: spanish
        expected:
          summary_language: spanish

  # Skills this skill depends on (optional)
  dependencies:
    - id: detect-language
      version: ">=1.0.0"
```

## Schema Reference

### Top-level fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `skill.id` | string | ✅ | Unique identifier. Kebab-case. Max 64 chars. Must match `[a-z0-9-]+` |
| `skill.version` | string | ✅ | Semantic version (MAJOR.MINOR.PATCH) |
| `skill.spec_version` | string | ❌ | The version of the ai-skills spec this file uses (e.g. `"0.1"`) |
| `skill.name` | string | ✅ | Human-readable display name. Max 100 chars |
| `skill.description` | string | ✅ | What the skill does. Max 1000 chars |
| `skill.author` | string | ❌ | Author username or org name |
| `skill.license` | string | ❌ | SPDX license identifier (e.g. MIT, Apache-2.0) |
| `skill.tags` | array | ❌ | List of searchable string tags |
| `skill.inputs` | array | ✅ | List of input parameter definitions |
| `skill.outputs` | array | ✅ | List of output definitions |
| `skill.execution` | object | ✅ | How the skill runs |
| `skill.compatible_with` | array | ❌ | Target frameworks |
| `skill.benchmarks` | object | ❌ | Performance data and test cases |
| `skill.dependencies` | array | ❌ | Other skills this skill uses |

### Input parameter fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | ✅ | Parameter name. Snake_case |
| `type` | string | ✅ | Data type (see Data Types below) |
| `description` | string | ❌ | What this parameter is for |
| `required` | boolean | ❌ | Default: `false` |
| `default` | any | ❌ | Default value when not provided |
| `enum` | array | ❌ | Allowed values |
| `minimum` | number | ❌ | Min value (for integer/number types) |
| `maximum` | number | ❌ | Max value (for integer/number types) |
| `min_length` | integer | ❌ | Min string length |
| `max_length` | integer | ❌ | Max string length |

## Execution Types

### `prompt`

Executes by sending a prompt to an LLM.

```yaml
execution:
  type: prompt
  model_hint: any           # Preferred model (optional)
  system_prompt: "..."      # System prompt (optional)
  prompt_template: "..."    # Uses {input_name} placeholders (required)
  output_parser: none       # none | structured | json
```

### `tool_call`

Executes by calling an external API or function.

```yaml
execution:
  type: tool_call
  function: my_module.my_function   # Python import path
  # OR
  endpoint:
    url: https://api.example.com/endpoint
    method: POST
    headers:
      Authorization: "Bearer {env.API_KEY}"
    body:
      query: "{input_text}"
```

### `chain`

Executes a sequence of other skills.

```yaml
execution:
  type: chain
  steps:
    - skill: detect-language
      input_map:
        text: "{input_text}"
      output_as: lang_result
    
    - skill: translate-text
      input_map:
        text: "{input_text}"
        target_language: english
      condition: "{lang_result.language} != english"
    
    - skill: summarize-document
      input_map:
        document: "{translate_text.translated_text}"
```

### `code`

Executes a Python code block directly.

```yaml
execution:
  type: code
  language: python
  runtime: ">=3.10"
  code: |
    def run(inputs):
        text = inputs["text"]
        word_count = len(text.split())
        return {"word_count": word_count, "char_count": len(text)}
```

| Type | Description | Example |
|------|-------------|---------|

## Versioning

### Skill Versioning

Skills use **Semantic Versioning** (semver.org):

- `MAJOR` — breaking change (input/output schema changed)
- `MINOR` — new feature, backwards compatible (new optional input)
- `PATCH` — bug fix, no schema change (improved prompt)

Once a version is published to the registry, it is **immutable**. You cannot overwrite a published version.

Dependency version constraints follow npm-style semver ranges:

```yaml
dependencies:
  - id: detect-language
    version: ">=1.0.0 <2.0.0"  # Compatible with 1.x
  - id: translate-text
    version: "^2.1.0"           # Compatible with 2.x, >= 2.1.0
```

### Spec Versioning

The `skill.yaml` format itself is versioned (e.g., `0.1`). Skills can optionally declare which version of the spec they target using the `skill.spec_version` field. 

The SDK and registry use this to ensure compatibility as the standard evolves:
- `0.x` — Initial draft phases. Breaking changes possible between minor versions.
- `1.x` — Stable release. Backwards-compatible additions only.

## Compatibility & Adapters

The ai-skills SDK automatically generates framework-specific code from a `skill.yaml`. This is done via **adapters**.

Each adapter takes a `skill.yaml` and outputs native code for that framework:

```
skill.yaml → [langchain adapter]  → LangChain Tool class
skill.yaml → [autogen adapter]    → AutoGen FunctionTool
skill.yaml → [crewai adapter]     → CrewAI Tool
skill.yaml → [raw_api adapter]    → Plain Python function
```

Adapters are pluggable — third parties can write adapters for new frameworks.

## Benchmarks

The `benchmarks` block is optional but strongly encouraged. It enables:

- Automated quality regression testing
- Registry-wide performance scoring
- Comparison between skill versions

### Benchmark fields

| Field | Type | Description |
|-------|------|-------------|
| `avg_latency_ms` | integer | Average execution time in milliseconds |
| `avg_cost_per_call_usd` | number | Average LLM/API cost per execution |
| `last_evaluated` | date | When benchmarks were last run |
| `test_cases` | array | List of test case definitions |

### Test case fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique test case ID |
| `description` | string | What this test checks |
| `input` | object | Input values for the skill |
| `expected` | object | Assertions on the output |

### Expected assertions

| Assertion | Description |
|-----------|-------------|
| `field_equals` | Output field equals exact value |
| `field_contains` | Output field contains substring |
| `field_type` | Output field is of expected type |
| `field_min_length` | String/array is at least N long |
| `field_max_length` | String/array is at most N long |
| `*_count_min` | Array has at least N items |
| `*_count_max` | Array has at most N items |

## Registry Protocol

The ai-skills registry is a public, open-source index of published skills.

### Publishing

```bash
aiskills publish
```

This:
1. Validates the `skill.yaml`
2. Runs all test cases
3. Records benchmark results
4. Uploads to the registry under `{author}/{skill-id}@{version}`

### Fetching

```bash
aiskills install jane-doe/summarize-document
aiskills install jane-doe/summarize-document@1.2.0   # Specific version
```

### Registry API

```
GET  /skills                         List all skills (paginated)
GET  /skills/{author}/{id}           Get latest version of a skill
GET  /skills/{author}/{id}/{version} Get specific version
GET  /skills/search?q={query}        Search skills by name/tag
POST /skills                         Publish a new skill (authenticated)
```
